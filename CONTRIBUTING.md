# Contribution Guidelines

Thank you for considering contributing to Chroniques d'Elaria! We appreciate your help in making this project better. Below are some guidelines to follow when contributing to this repository.

## Types of Contributions

1. **Bug Reports**: If you find a bug, please report it with a detailed description of how to reproduce it.
2. **Feature Requests**: We welcome new features! Please outline your suggested features with detailed use cases.
3. **Code Contributions**: Feel free to fork the repository, make changes, and submit a pull request.
4. **Documentation Improvements**: Help us improve our documentation by submitting issues or pull requests.
5. **Testing**: Help us enhance our test coverage by adding tests for new features or bugs.

## Contribution Process

1. **Fork the Repository**: Click the "Fork" button on the top-right corner of this repository to create your own copy.
2. **Clone Your Fork**: Run the following command to clone your fork:
   ```
   git clone https://github.com/<your-username>/Chroniques-d-elaria.git
   ```
3. **Create a Branch**: Create a new branch for your feature or bug fix:
   ```
   git checkout -b my-feature-branch
   ```
4. **Make Changes**: Implement your changes in your local repository.
5. **Commit Your Changes**: Commit your changes clearly and concisely:
   ```
   git commit -m "Description of changes"
   ```
6. **Push to Your Fork**: Push your changes back to your fork:
   ```
   git push origin my-feature-branch
   ```
7. **Open a Pull Request**: Go to the original repository and create a new pull request from your branch.

Thank you for your contributions! We look forward to collaborating with you!
